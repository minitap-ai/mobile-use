# GROWTH.md — Community Visibility Playbook

> *A guide for the mobile-use maintainers to increase project awareness and build a thriving contributor community.*

## 📍 Where mobile-use stands

**What makes mobile-use unique:**
- First open-source agent framework to achieve **100% on AndroidWorld benchmark**
- Natural language control for **both Android and iOS**
- Production-ready with Docker quick-start
- Backed by peer-reviewed research (arXiv paper)

**Current traction:** ~2.3k GitHub stars, active Discord community, growing PyPI downloads.

### Content Strategy

**What resonates with your audience:**

1. **Benchmark deep-dives** — Technical blog posts explaining how you achieved 100% accuracy
2. **Comparison posts** — "mobile-use vs Appium vs other mobile automation tools"
3. **Use case demos** — GIFs/videos showing real automation workflows
4. **Integration tutorials** — "Using mobile-use with Claude/GPT-4/Gemini"

**Where to post:**

| Platform | Content type | Frequency |
|----------|-------------|-----------|
| [r/MachineLearning](https://reddit.com/r/MachineLearning) | Research highlights, benchmark results | Monthly |
| [r/LocalLLaMA](https://reddit.com/r/LocalLLaMA) | Local LLM integration guides | Bi-weekly |
| [Hacker News](https://news.ycombinator.com) | Major releases, benchmark achievements | On milestones |
| [Dev.to](https://dev.to) | Tutorials, comparisons | Weekly |
| [X/Twitter](https://twitter.com) | Demo GIFs, release notes | 2-3x/week |

## 🤝 Community Building

### Contributor Experience

- [ ] Add `good first issue` labels to beginner-friendly tasks
- [ ] Create issue templates (bug report, feature request, device support)
- [ ] Document local development setup clearly
- [ ] Recognize contributors in release notes

### Discord Growth

- Create channels for different device types (#android, #ios-simulator)
- Weekly "Show & Tell" for community automations
- Pin troubleshooting guides for common issues
